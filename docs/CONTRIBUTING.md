# Panduan Kontribusi

Terima kasih telah berkontribusi pada proyek My-ISN. Kami menerapkan standar kualitas kode yang ketat untuk menjaga integritas ekosistem ERP ini.

## Standar Kode
- Seluruh kode baru harus mengikuti pedoman penulisan Dart/Flutter resmi.
- Gunakan arsitektur modular untuk setiap fitur baru.
- Hindari penggunaan warna *hardcoded*; gunakan `Theme.of(context)` untuk mendukung Light/Dark Mode.

## Alur Kerja (Workflow)
1. **Pengerjaan**: Buatlah branch baru untuk setiap fitur atau perbaikan.
2. **Review**: Setiap perubahan besar harus melalui tinjauan rekan setim (peer-review).
3. **Dokumentasi**: Perbarui file `docs/CHANGELOG.md` untuk setiap perubahan yang dilakukan sebelum penggabungan kode.

## Pelaporan Isu
Gunakan sistem manajemen isu internal untuk melaporkan bug atau mengajukan fitur baru dengan format yang telah ditentukan.
